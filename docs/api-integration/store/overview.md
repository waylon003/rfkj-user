# 门店模块总览

## 当前范围

- `src/pages/home/store-select.vue`
- `src/pages/home/store-detail.vue`
- `src/services/store/*`

## 当前状态

- 门店选择页已不再按“全门店列表”建模
- 当前页面模型已收口为“仅展示用户已注册门店”
- 首次登录若用户尚未注册任何门店会员，页面展示空状态并引导扫码注册
- 后端当前**缺少**“用户获取已注册门店列表”接口，因此本模块暂保持 mock facade

## 页面模型与当前能力

- `store-select.vue`
  - 展示范围：仅用户已注册门店
  - 排序规则：按定位距离最近优先
  - 卡片字段：门店名、地址、电话、距我 `x m / km`
  - 空状态：提示用户扫码注册成为门店会员
- `store-detail.vue`
  - 当前仍由本地 mock 详情承载
  - 与选择页统一按 `id` 跳转

## 缺失字段 / 接口

- 缺少接口：
  - 用户获取已注册门店列表接口
  - 用户已注册门店详情接口（若未来详情不再使用本地文案）
- 缺少能力：
  - 扫码注册成功后的门店列表刷新闭环
  - 真实门店图片、营业信息、介绍字段

## 接入策略

- 在后端补出“用户已注册门店列表”接口前，`services/store` 继续只做 mock/service facade
- 页面继续按当前前端模型收口，不回退到“全门店搜索”方案
- 若后端后续补接口，应优先在 `services/store` 做字段映射，不直接改页面消费结构
