# 后端接口反馈清单


---

## 一、已接入接口的字段补充需求

以下接口前端已接入，但返回数据缺少部分字段，页面功能无法完整闭环。

### 1. GET /wechat/member/userexpends — 礼品兑换记录

| 缺失字段 | 用途 | 说明 |
|---------|------|------|
| 提货地址 | 待领取礼品页面显示领取门店地址 | 当前前端写死"当前门店领取" |
| 核销码 / 二维码值 | 待领取礼品页面显示核销码弹窗 | 用户需出示核销码提货 |
| 过期时间 / 过期状态 | 待领取礼品页面区分"已过期"tab | 当前仅靠 `isTakeDelivery` 区分待领/已领，无法判断过期 |

### 2. GET /wechat/member/usertransfer — 互赠记录

| 缺失字段 | 用途 | 说明 |
|---------|------|------|
| 领取状态 | 区分已领取/未领取/已退回 | 当前只区分收入/支出，无法显示对方是否已领取 |
| 二维码值 | 查看二维码弹窗 | 赠送方需查看二维码让好友扫码领取 |
| 重发链接 | 未领取时重发领取链接 | 对方未领取时可重新发送 |

### 3. POST /wechat/member/usertransfer — 赠送提交

| 缺失字段 | 用途 | 说明 |
|---------|------|------|
| 赠送后二维码 | 赠送成功后展示领取二维码 | 用户需出示二维码让好友扫码领取 |
| 赠送后金额/数量 | 二维码弹窗中显示赠送数量 | 确认赠送信息 |
| 提示文案 | 二维码弹窗底部提示 | 如"请让好友使用微信扫一扫领取" |

### 4. POST /wechat/member/redeem — 彩票兑换

| 缺失字段 | 用途 | 说明 |
|---------|------|------|
| 兑换前预估积分 | 提交前显示预计获得积分数 | 用户提交前需要知道能换多少积分 |

## 二、完全缺失的接口需求

以下页面当前完全由 mock 驱动，需要后端提供对应接口。

### 1. 门店模块（高优先级）

| 接口 | 方法 | 说明 |
|------|------|------|
| 用户已注册门店列表 | GET | 返回用户已注册的门店列表，包含门店名、地址、电话、距离 |
| 门店详情 | GET | 返回单个门店的详细信息 |

### 2. 支付闭环（中优先级）

| 接口 | 方法 | 说明 |
|------|------|------|
| 购币下单 | POST | 自定义数量计价并创建订单 |
| 支付提交 | POST | 提交支付（微信支付等） |
| 支付结果查询 | GET | 查询订单支付状态 |
| 彩票抵扣规则 | GET | 查询彩票抵扣游戏币的规则 |
| 积分支付规则 | GET | 查询积分抵扣游戏币的规则 |

### 3. 活动模块（低优先级）

| 接口 | 方法 | 说明 |
|------|------|------|
| 活动列表 | GET | 返回活动列表，包含标题、时间、状态 |
| 活动详情 | GET | 返回活动详细信息 |
| 活动排行 | GET | 返回活动参与排行 |

### 4. 排行榜模块（低优先级）

| 接口 | 方法 | 说明 |
|------|------|------|
| 机台排名 | GET | 按机台类型（娱乐机/娃娃机/扭蛋机/弹珠机）维度的排名 |
| 我的排名 | GET | 当前登录用户的排名信息 |

> 现有 `GET /redeemtop` 只有积分兑换榜，页面需要机台/积分双 tab + 机型筛选，语义不匹配。
