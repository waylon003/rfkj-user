# 项目规则

## Figma 尺寸换算

- 在使用 Figma MCP 对齐设计稿时，凡是设计稿中读取到的 `px` 尺寸，默认按小程序 `rpx` 进行 `1px = 2rpx` 换算。
- 也就是说，Figma 中的任意 `px` 数值在实现到当前项目时，默认都需要先乘以 `2`。
- 只有在用户明确指出某个节点或属性不应按此规则换算时，才允许例外处理。

## TDesign 组件优先

- 当前项目使用 `tdesign-uniapp` 时，若 TDesign 组件库已有对应组件或图标，应优先使用 TDesign 组件/图标。
- 避免在已有成熟 TDesign 组件可用的情况下，直接使用原生 `uni-app` 基础组件重新实现同类能力。
- 本地组件文档优先查阅 `docs/tdesign-components.md`。
- 若本地文档信息不足，可再使用 Context7 查询 TDesign/tdesign-uniapp 的最新组件文档。

## 接口接入行为对齐

- 当任务属于“接口文档接入、真实接口联调、mock 迁移、service 适配”时，默认采用 `redeem-wxapp` 式行为：
  - 页面先按前端模型收口
  - 页面不直接消费后端原始结构
  - 接口字段变化优先在 `src/services/<domain>` 或 adapter 层处理
  - 只有当接口行为与页面模型冲突时，才考虑改页面模型
- 接入真实接口时，默认以页面模型为主；只有用户明确要求“接口字段为主”时，才反向按接口字段驱动页面模型。
- 接口行为一致但字段不全时，只允许在 service/adapter 层补空值、默认值或字段映射，不允许推导假业务数据。
- 对于“已决定真实接入”的页面或服务：
  - 不允许因为接口请求失败、接口返回空数据、或字段缺失，就在运行时静默回退到 mock 掩盖问题
  - 必须明确区分“哪些字段已接上、哪些字段缺失、哪些字段由前端模型补齐”
  - 若只缺少非关键字段且页面核心行为仍成立，可以继续真实接入，但必须把缺失字段登记到模块台账
  - 若缺少关键字段，导致页面核心列表、详情、提交流程或状态流转无法成立，则保持原 mock 服务，不做真实接口接入
  - 真实接入后的错误应显式暴露为真实接口问题，不能再用 mock 伪装成正常结果
- 接口行为不一致、能力缺失、列表/详情/表单语义不匹配时，不要强行接真；保留 mock 或页面现有契约，并记录为接口缺口。
- 只要真实接口已经接入，无论是部分接入还是完整接入，缺失字段、语义差异、派生字段、回退逻辑都必须登记到对应模块台账，不允许仅在代码里隐式消化。
- 任何涉及真实接口接入的工作，开始前先看 `backend-api-integration-checklist.md`，并同步对应模块台账或 `docs/接口接入判断台账.md`。
- 接入完成后，必须同步更新根目录总台账和对应模块文档，保持“文档先行、代码跟随”的节奏。
